# `KS_KG/models/` — picker-model runs (stead vs original)

This directory adds a **picker-model dimension** to the KS_KG seismicity pipeline so the
existing `stead` results and a new `original` run can coexist without colliding.

> **Two different "models" — don't confuse them:**
> - **Picker model** (this directory): the PhaseNet weights — `stead` vs `original`.
> - **Velocity model** (`kim1983` / `kim2011`): the crustal model used by PyOcto / HYPOINVERSE.
>   These live *inside* each picker-model tree, exactly as before.

Everything here was generated by [`build_original_tree.py`](build_original_tree.py), which
writes **only** under `KS_KG/models/` and never modifies any pre-existing file. Re-running it
regenerates the `original/` copies; the originals stay untouched.

## Layout

```
KS_KG/
  continuous/  station_table/  velocity_model/        # SHARED inputs (untouched)
  picks/  pyocto/  HypoInv/  detection_location/       # the existing "stead" run (untouched)

  models/
    build_original_tree.py        # regenerates this tree (idempotent, writes only here)
    stead/                        # read-only symlinks to the existing run, for side-by-side navigation
      detection_location -> ../../detection_location
      picks/ pyocto/ HypoInv/ station_table  (all symlinks)
    original/                     # the new run — copies of the notebooks, switched to "original"
      detection_location/<year>/  01.Run_multi-station_detection_<year>.ipynb   # model -> "original"
                                   02.PyOcto_association_<year>.ipynb            # paths -> original tree
                                   03.Plot_association_results_<year>.ipynb      # reads original pyocto
                                   picks/                                        # (created on run)
      pyocto/                      # pyocto_kim1983_<year>.csv + assignment land here
      station_table/              # association side-writes (stations_<year>.csv) land here, NOT shared
      HypoInv/
        04.Make_input_PHS_file_for_HypoInv.ipynb       # reads original pyocto, writes original PHS
        UF<year>.sh                                    # verbatim copy; relative paths resolve here
        PHS/                                           # UF<year>.phs land here
        STA            -> ../../../HypoInv/STA          # symlink (station files are picker-independent)
        kim1983/  kim2011/                              # real dirs; *_p.crh/*_s.crh symlinked in,
                                                        #   UF<year>.{sum,prt,arc} outputs land here
      hypodd/                                          # placeholder for future relative relocation
```

## What was redirected when copying `stead` → `original`

| Notebook | Edit applied to the copy |
|---|---|
| `01.Run_multi-station_detection` | `from_pretrained("stead")` → `"original"`; legacy `2014_sequence/continuous` → `KS_KG/continuous` (2010, 2013 only) |
| `02.PyOcto_association` | picks input, `pyocto/` output, and `stations_<year>.csv` side-write → `models/original/…`; `stations.csv`, `station_update.dat`, `velocity_model/kim1983.csv` left pointing at the **shared** originals (read-only) |
| `03.Plot_association_results` | `pyocto/` reads → `models/original/pyocto/` |
| `04.Make_input_PHS_file_for_HypoInv` | `pyocto/` input + `HypoInv/PHS/` output → `models/original/…` |
| `UF<year>.sh` | copied verbatim — run it from `models/original/HypoInv/` so its relative `STA/`, `<model>/`, `PHS/` paths resolve into this tree |

## How to run the `original` pipeline (per year)

1. **Detection** — open `original/detection_location/<year>/01.Run_multi-station_detection_<year>.ipynb`, run it. Picks land in `original/detection_location/<year>/picks/`.
2. **Association** — run `02.PyOcto_association_<year>.ipynb`. Events/assignments land in `original/pyocto/`.
3. **PHS file** — open `original/HypoInv/04.Make_input_PHS_file_for_HypoInv.ipynb`, set `year=<year>`, run. Produces `original/HypoInv/PHS/UF<year>.phs`.
4. **Location** — `cd original/HypoInv && sh UF<year>.sh`. The `model=` line inside selects the velocity model (`kim1983`/`kim2011`); outputs land in `original/HypoInv/<model>/UF<year>.{sum,prt,arc}`.
5. **Compare** with stead: the matching files are under `models/stead/…` (symlinks to the existing run).
