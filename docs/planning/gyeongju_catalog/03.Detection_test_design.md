**Associators** (each consumes all 4 pick sets identically):

**Fairness rules** (pre-registered):
- identical station set + month for every combo; NS decimated to 100 Hz with proper anti-alias
  (lowpass 45 Hz → decimate ×2) for the standard matrix (native-rate handled separately in 03e);

**Metrics** (03d): KMA-event recall (match: origin ±5 s, epicentre ≤30 km); total/new detections + daily rate
curves; fraction of picks associated; per-event phase counts + azimuthal-gap distributions; cross-method
event-agreement matrix (8×8); wall-clock runtime per combo. Verdict table + take-homes computed in-notebook.

## 3 · The 200 Hz native-rate strategy — feasible, one line, must be benchmarked (03e)

**Mechanics (verified on this machine, seisbench 0.11.0):** SeisBench models are fully convolutional — they
judge in **samples**; the "100 Hz" is metadata used by `annotate()` to resample input. Overriding
`model.sampling_rate = 200` makes `annotate()` consume 200 Hz natively (no resample) and timestamp on the native
5-ms grid. Verified: default run outputs 100 Hz probability traces (internal resample), override outputs 200 Hz
with correct absolute times.

**Trade-offs to test, stated up front:**
- receptive field halves in real time: 3001 samples = 15 s at 200 Hz (S–P at 100 km ≈ 12–13 s → still inside,
  but P and S of distant events sit near window edges more often);
- the model "hears" the waveform pitch-shifted: a 20 Hz onset looks like a 10 Hz onset in normalized frequency —
  usually benign for broadband-trained PhaseNet, but this is exactly the empirical question;
- benefit: no anti-alias filtering of the 40–100 Hz microseismic energy + picks on the 5-ms grid.

**Benchmark (03e), on real event waveforms:** take KMA events of 2017-12; cut 60-s NS windows (raw 200 Hz).
Treatments: **T1** proper decimate→100 Hz (anti-alias) · **T2** native-200 (`sampling_rate=200`) ·
T3 (optional, cautionary) naive resample without explicit anti-alias. Metrics: recall vs predicted arrivals
(kim2011 1-D, ±0.5 s), false-pick rate on pre-event noise windows, pick-time offset distribution T2−T1
(median/IQR), pick-probability distributions — all SNR-stratified; plus a 50-pick visual gallery for eyeballing.
**Pre-registered decision rule:** adopt native-200 for the final run iff recall ≥ T1 and offset IQR ≤ T1
(no degradation ⇒ take the precision/fidelity benefit).
