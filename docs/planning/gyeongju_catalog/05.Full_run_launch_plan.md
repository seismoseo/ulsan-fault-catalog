# Full 2010–2026 detection run — launch plan (drafted 2026-07-19)

## Pipeline (settled + validated on 2021-09 dense)
**PhaseNet+ (GPU-fixed, 3 day-shards, NS pre-decimated to 100 Hz for detection) → PyOcto (kim2011, fixed 4/2/2,
daily-chunked) → [downstream] HYPOINVERSE location-QC → HypoDD.** Recall 43/43 UF + 24/24 KMA (1 formal, a 6 s
origin offset). Detection is now ~1.0 s/station-day (faster than PhaseNet-original).

## Runtime budget (measured)
- Detection: **74 min / dense month** (241 sta, 3-shard). ~9 dense years × 12 ≈ **~5.5 days**.
- Association: 66 min/month sequential (PyOcto is internally multi-threaded → day-pools oversubscribe; **overlap it
  with detection on the CPU** while the GPU detects the next month → hides under detection wall-clock).
- Sparse years (2010–2016, ~20–40 sta, no NS) are far cheaper (~1–2 days total).
- **Full detection+association ≈ 5.5–7 days on one GPU — inside the 1-week target.** Location/relocation are
  separate downstream stages.

## THE PREREQUISITE — local data completeness (current blocker)
Local waveform coverage today (from nb02 + disk scan):
- **NS**: 2017–2023 only (NS did not exist pre-2017; **2024–2026 NOT downloaded**). 394,177 confirmed-missing
  station-days within the span.
- **KS**: gaps throughout (173,778 missing station-days); local KS = NECIS cron archive, currently ~2026 only +
  the stage-1 KS_KG/continuous UF subset. **Pre-2015 KS epochs still unqueried (NECIS).**
- **KG**: 62,255 missing station-days.
So a COMPLETE 2010–2026 run needs a gap-fill pass first (NECIS for KS + KG history; GHBSN holder for NS 2024–2026 &
pre-2017.332). This matches the user's stated order: *fill local gaps → then run detection*.

## Disk
NS 100 Hz mirror = **146 GB / month** (2021-09 measured). Bulk mirror for 2017–2023 ≈ **~10 TB** (61 TB free — OK).
RECOMMENDED: **rolling decimation** — `predecimate_ns.py` per month right before detecting it, delete the mirror
after that month's picks are written. Keeps peak disk ~150 GB, not 10 TB. (Original 200 Hz NS/ always preserved.)

## Full-run architecture (per-month driver, to build)
```
for each month M in 2010-01 .. 2026-06:
  build_stations.py --month M                      # per-era station set (auto: KS/KG always, NS 2017+, GJ 2016-17)
  predecimate_ns.py --month M                      # rolling 100 Hz mirror (skip if no >100 Hz station)
  run_pnplus_month.py --month M --predecimated --doy-start/--doy-end  (x3 shards)  ->  assemble parquet
  associate_daily.py --month M                     # PyOcto 4/2/2  (overlap with next month's detection)
  rm -rf NS_100hz/<that month>                     # rolling cleanup
```
Detection has per-station checkpointing (crash-safe/resumable); the driver should be month-level resumable
(skip months whose pick parquet already exists). Pipeline detection(GPU) ‖ association(CPU) across months.

## Downstream (separate, after detection+association)
Location = HYPOINVERSE with the consistent physical QC (Ulsan-style gap/num or Buan v3 RMS<1.0/ERH<5/ERZ<5/
gap<180/num>7) — THIS is where the 4/2/2 over-detection (~10k raw events/month) is filtered to real events, applied
identically to every year → temporally consistent completeness. Then HypoDD relocation (dt.ct + dt.cc).
